01 — The Challenge
🏏

The Problem

Scouts lacked a modern mobile platform for athlete videos, ratings, and performance workflows. Existing scouting was manual, fragmented, and slow.

  • No centralised platform to track athlete progress over time
  • Video highlights scattered across WhatsApp and USB drives
  • No structured rating system for batting, bowling, and fitness
  • Coaches and parents had no real-time performance visibility
  • Draft day decisions made on gut feel, not structured data
🎯

The Solution

I built Scoutflix as a Compose-first Android product for cricket scouting — video highlights, athlete ratings, and real-time coach workflows.

Solo Android ownership included architecture, ExoPlayer media integration, and Firebase-backed delivery.

  • Athlete profiles with 360° performance ratings across 4 dimensions
  • Video highlight playback via ExoPlayer / Media3
  • Session planning and coach feedback tools
  • Paged athlete feed with scout shortlist and draft management
  • Real-time push notifications via Firebase Cloud Messaging
02 — How It Was Built

Architecture first.
Then every feature.

Scoutflix was built with a modern, fully Compose-first approach — no legacy XML, no legacy architecture. Every layer was designed for the complexity of real-time media, large paginated datasets, and multi-role users (athlete, coach, scout).
🏗️
Phase 01 · Foundation
Compose Architecture & Auth
Set up the full Jetpack Compose project with Hilt DI, Compose Navigation with type-safe routes, and MVVM + Repository pattern. Firebase Auth integration for multi-role sign-in — athlete, coach, and scout accounts each with different access levels.
Jetpack ComposeHiltNavigation ComposeFirebase Auth
👤
Phase 02 · Athlete Profiles
Performance Rating System & Athlete DNA
Built the core athlete profile system — multi-dimensional ratings for batting, bowling, fielding, fitness (strength/agility/speed/power/endurance), and mental attributes. Data persisted via Retrofit REST API and cached with DataStore Preferences for offline-first viewing.
RetrofitDataStoreCompose ChartsRating Bar
🎬
Phase 03 · Video
Video Highlights with ExoPlayer / Media3
Integrated AndroidX Media3 ExoPlayer into the Compose UI for smooth in-feed video highlight playback. Handled adaptive streaming, buffer management, lifecycle-aware playback, and custom Compose player controls. Coil 3 used for video thumbnails and image loading.
Media3 ExoPlayerMedia3 UICoil 3OkHttp
📄
Phase 04 · Discovery
Paged Athlete Feed & Scout Search
Built the paginated athlete discovery feed using Paging 3 + paging-compose. Scouts can filter by position, age, region, and rating. Shortlist management and draft-day report generation. Paging 3's LazyPagingItems integrated cleanly into Compose LazyColumn.
Paging 3paging-composeLazyColumnKotlinx Serialization
💬
Phase 05 · Communication
Coach Feedback, Session Planning & FCM Notifications
Built the coach-to-athlete feedback system, session planning screens, and parent visibility portal. Firebase Cloud Messaging (FCM) for real-time push notifications — session updates, rating changes, and scout interest alerts. Firebase Analytics and Crashlytics for production monitoring.
Firebase FCMFirebase AnalyticsFirebase Crashlytics
