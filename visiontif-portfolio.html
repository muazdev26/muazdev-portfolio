01 — AiVision Problem & Solution
👁️

The Problem

Blind users lack an affordable mobile tool to read text, recognise objects, and navigate without a sighted helper. Existing options were either expensive hardware or apps with poor accessibility design.

The client also needed external USB camera support for wearable and desktop assistive setups.

  • No affordable, accessible tool to read printed text hands-free
  • Object and scene recognition too slow or inaccurate on-device
  • No support for external USB cameras for non-mobile setups
  • Existing apps not built with TalkBack accessibility in mind
  • Barcode scanning for product identification missing or unreliable
🤖

The Solution — AiVision

I built AiVision as an accessible Android assistant that reads text, describes scenes, and scans barcodes with TalkBack-friendly flows.

Delivered to Google Play Internal Testing for client review and accessibility validation.

  • Real-time text recognition in 60+ languages via ML Kit OCR
  • Azure AI Vision for rich scene descriptions spoken aloud
  • Custom TFLite model for on-device object detection (offline)
  • External USB camera support via libusb + UVC protocol
  • Barcode & QR scanning for product identification
  • Full TalkBack / screen reader accessibility on all screens
  • Google Sign-In with accessible credential flow
  • SQLDelight for scan history with offline access
AiVision — Features

Turning a phone camera into
a set of intelligent eyes.

Every feature reads back to the user through audio — no sight required to operate anything.

📖

Real-Time Text Recognition — ML Kit + Azure AI

Instantly reads printed text from signs, menus, documents, and handwriting using Google ML Kit OCR with support for 60+ languages. For complex documents, Azure AI Vision's cognitive API provides layout-aware extraction — paragraphs, tables, and structured content — all spoken aloud with full TalkBack compatibility.

🔍

On-Device Object Detection

Custom TFLite models + ML Kit Object Detection identify objects in the camera frame in real time — entirely offline. No internet needed for core detection features.

🌍

Scene Description — Azure AI

Azure AI Vision analyses the full camera frame and returns a natural language scene description — "A busy café counter with two people in line" — spoken back immediately.

🔌

External USB Camera (libusb + UVC)

Integrated libusb and UVC camera protocol so the app works with external cameras — wearable glasses, desk mounts, or assistive hardware — not just the phone's built-in camera.

📦

Barcode & QR Scanning

ML Kit Barcode Scanning identifies products from barcodes and QR codes — product name, description, and details are read aloud. Critical for blind grocery and retail independence.

♿

Full Accessibility Support

Every screen built from the ground up for TalkBack compatibility — content descriptions on all interactive elements, logical focus order, haptic feedback, and voice-first navigation throughout.

🔐

Google Sign-In + Credentials API

Accessible authentication via Google Sign-In using AndroidX Credentials API and Google Identity Library — simple, screen-reader-friendly one-tap login.

📂

Scan History — SQLDelight

Every scan stored locally with SQLDelight. Users can replay previous text reads, object detections, or scene descriptions without an internet connection.

🧠

OpenCV Integration

OpenCV used for pre-processing camera frames — contrast enhancement, perspective correction, and image normalisation before passing to ML models for higher accuracy.

02 — AiNavigation Problem & Solution
🗺️

The Problem

Blind visitors had no indoor audio wayfinding tool for galleries and cafés. GPS was too imprecise, and venues needed a hands-free navigation alternative.

The client also required a solution that worked with existing BLE beacon infrastructure.

  • GPS doesn't work indoors — too imprecise for room-level navigation
  • No audio wayfinding system for gallery or café spaces
  • Blind visitors required sighted companions to navigate
  • Staff had no way to push contextual announcements to visitors
  • Needed to work offline — no reliable venue Wi-Fi assumed
🦋

The Solution — AiNavigation

I co-developed AiNavigation in Flutter to turn BLE beacon location into audio wayfinding and venue guidance.

Delivered as a cross-platform prototype through TestFlight and Firebase App Distribution.

  • BLE beacon scanning for real-time indoor positioning
  • Text-to-speech narration via flutter_tts for all navigation cues
  • Cloud Firestore for real-time venue map and exhibit data
  • Firebase App Distribution + TestFlight delivery
  • Offline-capable with shared_preferences local state
  • Location permission handling via permission_handler
  • Wake lock to keep screen active during navigation sessions
AiNavigation — Features

Indoor navigation,
entirely through audio.

No screens required. The app speaks everything — position, exhibits, directions, and announcements.

📡

BLE Beacon Indoor Positioning

Uses dchs_flutter_beacon to continuously scan for Bluetooth Low Energy beacons installed throughout the gallery and café. RSSI signal strength determines the user's current zone — no GPS or Wi-Fi needed. Position updates trigger contextual audio narration automatically as the user moves through the space.

🔊

Text-to-Speech Navigation — flutter_tts

All navigation cues, exhibit descriptions, and directional instructions delivered through the device speaker via flutter_tts. No interaction required from the user — walk and listen.

☁️

Real-Time Venue Data — Cloud Firestore

Gallery layout, exhibit descriptions, and café items stored in Cloud Firestore. Venue staff can update content in real time without an app update — changes propagate instantly to all visitors.

📵

Offline-First with SharedPreferences

Last-known venue map and exhibit data cached locally with shared_preferences. Core navigation works even when the venue has unreliable connectivity.

💡

Wake Lock — Screen Always On

wakelock_plus keeps the device screen active during navigation sessions — so audio narration never cuts out because the phone went to sleep mid-gallery walk.

🔔

Fluttertoast & Audio Alerts

Non-intrusive toast alerts and audioplayers for sound cues at key navigation points — entry detection, exhibit proximity, and route completion confirmations.
